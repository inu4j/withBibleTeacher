## 1. 프로젝트 개요

### 1.1 프로젝트 비전

> **"차가운 정보(Cold Information)는 기술(AI)에게 위임하고, 사역자는 따뜻한 정보(Warm Information)에 전념한다"**

WITH Bible 프로젝트는 선교지의 만성적인 '자료의 기근(Famine of Resources)' 문제를 해결하기 위해 생성형 AI 기술을 활용한 맞춤형 성경공부 교재 생성 플랫폼을 개발하고, 이를 통해 선교사/교사가 교재 준비에 소모하던 시간을 목양적 돌봄에 재투자할 수 있도록 하는 것을 목표로 합니다.

### 1.2 프로젝트 이중 목표

| 구분 | 목표 | 산출물 |
|------|------|--------|
| **학술 연구** | D.Min 박사학위 논문 완성 | 논문 + 학술적 검증 |
| **실제 개발** | WITH Bible 웹 플랫폼 개발 및 배포 | 웹 서비스 + 오픈소스 |

### 1.3 핵심 개념: 정보의 이원화 (Information Dichotomy)

```
┌─────────────────────────────────────────────────────────────────┐
│                      정보의 이원화 구조                          │
├─────────────────────────┬───────────────────────────────────────┤
│    차가운 정보 (Cold)    │       따뜻한 정보 (Warm)              │
├─────────────────────────┼───────────────────────────────────────┤
│ • 데이터, 텍스트, 논리   │ • 관계, 공감, 영적 감화               │
│ • 형식적 지식(Explicit)  │ • 암묵적 지식(Tacit Knowledge)        │
│ • 기능적, 효율성 기반    │ • 인격적, 변화/회심 지향              │
│ • AI가 탁월하게 처리     │ • 인간 사역자만의 성역                │
├─────────────────────────┴───────────────────────────────────────┤
│  → AI로 차가운 정보 효율화 → 따뜻한 정보를 위한 시간 확보         │
│  → "High Tech를 통해 High Touch를 위한 시간을 산다"               │
└─────────────────────────────────────────────────────────────────┘
```

### 1.4 WITH Bible 명칭 의미

- **W**orld **I**nnovation **T**echnology **H**ub
- 기술(IT)이 말씀(Bible)과 함께(With) 선교를 돕는다

### 1.5 대상 사용자

| 구분 | 대상 |
|------|------|
| 1차 대상 | 태국 내 미얀마 이주민 교회 주일학교 교사 |
| 2차 대상 | 자료 부족 선교지의 모든 기독교 교육 사역자 |

#### 2.2.5 검증 지표의 객관화

**지적사항:** 교사의 자가 보고만으로는 신뢰도 부족

**신학적 정확성 블라인드 테스트:**

1. AI 생성 교재 30개 + 인간 작성 교재 20개 = 총 50개

## 4. RAG 시스템 설계 및 데이터베이스 구축

### 4.1 RAG 시스템 개요

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         RAG 시스템 처리 흐름                             │
└─────────────────────────────────────────────────────────────────────────┘

[사용자 쿼리] "창세기 1장 1절을 7세 아이에게 가르치는 교재 만들어줘"
        │
        ▼
┌───────────────────┐
│   Query Analysis   │  • 성경 구절 파싱: 창세기 1:1
│   쿼리 분석        │  • 대상 연령: 7세
└─────────┬─────────┘  • 언어: 한국어/미얀마어
          │
          ▼
┌───────────────────┐
│    Embedding      │  • 쿼리 → 벡터 변환
│    임베딩 생성     │  • Model: multilingual-e5-large
└─────────┬─────────┘
          │
          ▼
┌───────────────────────────────────────────────────────────┐
│                    Vector Database                        │
│  • 관련 주석 검색 (창세기 1:1 관련)                         │
│  • 유사 교재 검색                                          │
│  • Top-K (K=5) 결과 반환                                   │
└─────────────────────────┬─────────────────────────────────┘
                          │
                          ▼
┌───────────────────────────────────────────────────────────┐
│                   Context Assembly                        │
│  검색된 문서 + 원본 쿼리 + 시스템 프롬프트 조합             │
└─────────────────────────┬─────────────────────────────────┘
                          │
                          ▼
┌───────────────────────────────────────────────────────────┐
│                 LLM Generation (Gemini)                   │
│  조합된 컨텍스트 기반 교재 생성                             │
└─────────────────────────┬─────────────────────────────────┘
                          │
                          ▼
┌───────────────────────────────────────────────────────────┐
│                    Post-processing                        │
│  • 출력 포맷팅                                            │
│  • 환각 체크 (성경 구절 검증)                              │
│  • 이미지 생성 트리거                                      │
└───────────────────────────────────────────────────────────┘
```

#### 4.3.1 시스템 프롬프트 핵심 요소

```markdown
# WITH Bible 교재 생성 시스템 프롬프트

## 역할 정의
당신은 어린이 기독교 교육 전문가이자 선교적 상황화 전문가입니다.

## 신학적 원칙
1. 성경의 권위: 성경을 하나님의 말씀으로 존중
2. 복음 중심: 예수 그리스도의 복음을 중심에
3. 신학적 균형: 특정 교파 편향 없이 복음주의적 관점
4. 검증된 해석: 제공된 주석과 교리 자료를 우선 참조

## 교육학적 원칙 (피아제 기반)
- 4-6세 (전조작기): 구체적 사물, 간단한 문장, 많은 그림
- 7-9세 (구체적 조작기): 논리적 연결, 실제 예시, 활동 중심
- 10-14세 (형식적 조작기): 추상적 개념, 토론, 적용 확장

## 상황화 원칙
1. 문화적 예화: 미얀마/태국 문화에서 친숙한 이미지
2. 언어 적합성: 해당 연령이 이해할 수 있는 어휘
3. 시각적 표현: 아시아 문화권에 적합한 인물 묘사

## 금지 사항
- 성경 본문 왜곡 또는 임의 수정
- 검증되지 않은 해석 제시
- 아동에게 부적절한 표현
```

### 4.4 환각 방지 전략

```
┌─────────────────────────────────────────────────────────────────┐
│                    환각 방지 다층 검증                           │
└─────────────────────────────────────────────────────────────────┘

Layer 1: RAG 기반 참조
├── Vector DB에서 검증된 자료만 참조
├── 출처 명시 의무화
└── "모르면 말하지 않기" 원칙

Layer 2: 규칙 기반 검증
├── 성경 구절 정확성 체크 (DB와 대조)
├── 신학적 키워드 필터링
└── 금지 표현 목록 체크

Layer 3: 후처리 검증
├── 생성된 성경 인용문 자동 검증
├── 교리적 일관성 체크
└── 논리적 모순 탐지

Layer 4: HITL (Human-in-the-Loop)
├── 교사의 최종 검토
├── 수정/거부 권한
└── 피드백 수집 → 시스템 개선
```

### 재귀적 집필 계획 (Recursive Writing Cycle)
1. **분석:** 해당 섹션의 핵심 논지와 참고 자료(프로포잘, 피드백 등) 분석.
2. **집필:** 2단계 소제목 단위로 목표 분량에 맞춰 상세 집필 (Markdown 형식).
3. **저장:** 해당 챕터 MD 파일에 내용 업데이트.
4. **Git:** 작업 내용 커밋 및 푸시 (자동화).
5. **보고:** 완료 보고 및 다음 섹션 예고.
