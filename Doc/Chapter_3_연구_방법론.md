# 제 3 장 연구 방법론 (Methodology)

본 장에서는 'WITH Bible' 시스템의 개발 과정과 이를 태국 내 미얀마 이주민 주일학교 현장에 적용하여 그 효과성을 검증하는 구체적인 연구 절차를 기술한다. 본 연구는 "자원 부족 선교지의 교육 역량 강화"라는 실천적 목표를 달성하기 위해, 개발 연구(Design-Based Research)와 사례 연구(Case Study)를 결합한 혼합 연구 방법(Mixed Methods)을 채택한다.

## 3.1. 프로젝트 설계 : 보편적 상황화 도구 'WITH Bible' (Project Design)

본 연구의 핵심 산출물인 'WITH Bible'은 특정 지역에 국한되지 않는 '범용적 선교 교육 플랫폼(Universal Mission Education Platform)'을 지향한다. 시스템 설계의 핵심 원칙은 신학적 안전성, 문화적 적절성, 그리고 사용자 접근성이다.

### 3.1.1. RAG (검색 증강 생성) 기반의 신학적 가드레일 구축
인공지능의 한계인 환각(Hallucination) 현상을 방어하고 선교지의 신학적 정체성을 유지하기 위해 RAG(Retrieval-Augmented Generation) 기술을 시스템의 근간으로 삼는다. 
첫째, **신학적 데이터베이스(Vector DB) 구축**: 개혁주의적 관점의 성경 주석, 교리 문답, 그리고 검증된 성경공부 교재 텍스트를 수집하여 벡터화한다. 특히 영미권 자료뿐만 아니라 아시아적 맥락을 담은 신학적 텍스트를 의도적으로 포함시켜 알고리즘적 식민주의를 방어한다.
둘째, **적응형 생성 파이프라인**: AI가 교재를 생성할 때 시스템은 먼저 구축된 벡터 DB에서 가장 연관성 높은 신학적 근거를 실시간으로 검색(Retrieval)한다. 검색된 데이터를 바탕으로 AI가 답변을 생성(Generation)하게 함으로써, 임의의 해석을 배제하고 사역자가 신뢰할 수 있는 범위 내에서 콘텐츠를 생산하도록 강제한다.

### 3.1.2. 교육학적·선교학적 맞춤형 모듈 설계
생성된 콘텐츠가 학습자에게 실제적인 효용을 갖도록 두 가지 적응 모듈을 탑재한다.
첫째, **발달 단계별 변환 모듈**: 피아제의 인지 발달 단계에 따라 어휘의 난이도와 활동지의 복잡성을 조절한다. 예를 들어 5세 아동용 교재는 구체적 사물 중심의 짧은 문장과 삽화를, 13세 청소년용은 추상적 질문과 토론 위주의 텍스트를 자동 생성하도록 프롬프트를 정교화한다.
둘째, **문화적 상황화 이미지 모듈**: 생성형 AI의 이미지 생성 기능을 연동하여, 성경의 인물과 배경을 동남아시아적 상황에 맞게 시각화한다. 금발의 백인이 아닌 미얀마·태국 아동들에게 친숙한 인종적·문화적 특징을 가진 삽화를 제공함으로써 복음의 수용성을 극대화한다.

### 3.1.3. '인간 검증(HITL)' 워크플로우를 통한 사역적 권위 보존
기술이 사역자의 자리를 찬탈하지 않도록 Human-in-the-Loop(HITL) 방식을 시스템 UI에 내재화한다. AI가 교재를 생성한 후 즉시 확정되지 않고, '사역자 검토 모드'를 반드시 거치게 한다. 사역자는 AI가 제안한 내용을 읽고 수정, 편집하거나 삭제할 수 있으며, 최종 승인(Approve) 버튼을 눌러야만 인쇄용 PDF가 생성된다. 이 과정은 사역자가 AI를 하부 노동자로 부리는 '영적 주권'을 시각적으로 확인하게 하며, 기술 사용이 영적 위탁이 아닌 사역자의 분별력을 훈련하는 과정이 되도록 설계한다.
